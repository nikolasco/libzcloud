Getting Module Information¶

Among other things, zcloud_init gathers basic information about the available plugins. It does this in such a way that those plugins are not actually loaded, so it is quick and does not require that all dependencies of all plugins be available.

Information on available plugins is available from either of these functions:

ZCloudStorePlugin *zcloud_get_store_plugin_by_prefix(const gchar *prefix)¶

Get the plugin object for the given prefix.

Parameters:
  • prefix – desired storage specification prefix
Returns:

a plugin object, or NULL if not found
GSList *zcloud_get_all_store_plugins(void)¶

Get a list of all plugins.

Returns:a GSList of pointers to ZCloudStorePlugin objects

Plugin Information¶

ZCloudStorePlugin¶

Information about a single libzcloud plugin

gchar *ZCloudStorePlugin.prefix¶
The prefix for this plugin
ZCloudModule *ZCloudStorePlugin.module¶
Information about the module that will define this plugin
GType ZCloudStorePlugin.type¶
The type of the store class that will be instantiated for this plugin
GPtrArray *ZCloudStorePlugin.paramspecs¶
A list of GParamSpec objects describing the parameters for this plugin. The GLib documentation has the complete story on these objects. Briefly, the parameter name is availble in spec.name, the expected type is in spec.value_type, and the nick and blurb are available via g_param_spec_nick(spec) and g_param_spec_blurb(spec), respectively.

Module Information¶

ZCloudModule¶

Information about a module implementing plugins

gchar *ZCloudStorePlugin.basename¶
Base name of this module
gchar *ZCloudStorePlugin.module_path¶
Full pathname to the loadable shared object
gchar *ZCloudStorePlugin.xml_path¶
Full pathname of the XML file defining this module
gboolean ZCloudStorePlugin.loaded¶
True if this module has been loaded
