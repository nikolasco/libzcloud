Store¶

class ZCloudStore¶

Stores represent a service that provides persistence of arbitrary blobs. An example is Amazon S3.

All of the methods listed below will return true on success and false on failure. If they return false, they will set the error pointer to a GError object (assuming one was provided).

create(key, progress, error)¶

Creates the key with an empty value. This is not required by all store classes. See Uploading and Downloading.

Parameters:
  • key (String) – the key to create
  • progress – an optional ProgressListener
  • error – optional storage for an error
Returns:

true (on success) or false (on failure)
Return type:

boolean
upload(key, producer, progress, error)¶

Upload some data. producer will be consulted for metadata (e.g. MD5 hash and size) and is responsible for actually reading the data to be uploaded. The Store handles actually uploading the data to the storage service. See Uploading and Downloading.

Parameters:
  • key (String) – the key to store the data at
  • producer – an UploadProducer
  • progress – an optional ProgressListener
  • error – optional storage for an error
Returns:

true (on success) or false (on failure)
Return type:

boolean
download(key, consumer, progress, error)¶

Download data from key. The consumer is responsible for “doing something useful” with the data received from the storage service. For example, it could save it to a file. See Uploading and Downloading.

Parameters:
  • key (String) – the key to fetch data from
  • consumer – a DownloadConsumer
  • progress – an optional ProgressListener
  • error – optional storage for an error
Returns:

true (on success) or false (on failure)
Return type:

boolean
exists(key, progress, error)¶

Checks if key exists. See Testing for Existence.

Parameters:
  • key (String) – the key to check
  • progress – an optional ProgressListener
  • error – optional storage for an error
Returns:

true (on success) or false (on failure or non-existence)
Return type:

boolean
delete(key, progress, error)¶

Delete whatever is represented by key. See Deleting Keys.

Parameters:
  • key (String) – the key to delete
  • progress – an optional ProgressListener
  • error – optional storage for an error
Returns:

true (on success) or false (on failure)
Return type:

boolean
list(template, consumer, progress, error)¶

List all keys, calling ListConsumer.got_result() for each one. If a template is provided, only keys that fit it will be reported. The template should use %s as a placeholder/wildcard and %% to represent %. See Listing Keys.

Parameters:
  • template – the template to list matches for
  • consumer – a DownloadConsumer
  • progress – an optional ProgressListener
  • error – optional storage for an error
Returns:

true (on success) or false (on failure)
Return type:

boolean
